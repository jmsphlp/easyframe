# EasyFrame

Add clean borders to photos and videos without cropping. Free, fast, private — runs entirely in your browser. **Installable as a PWA on iOS, Android, and desktop.**

Inspired by [Simple Border](https://simpleborder.app/), built as a zero-cost web app instead of a native iOS app (no Apple Developer fee needed).

## What it does

- Photo and video borders without cropping (adds pixels instead)
- Aspect ratios: Original, 1:1, Instagram Portrait (4:5), Instagram Landscape (1.91:1), 9:16, 16:9, 3:2, 2:3
- Custom border color (presets + native color picker)
- Variable border thickness
- Batch photo export with clickable previews; videos are single-file only
- Exports PNG/JPEG photos and MP4/WebM videos with quality presets
- Web Share API on iOS so you can save straight to Photos
- Installable to home screen
- Works offline once installed
- 100% client-side — your files never leave your device

## Run locally

```bash
npm install
npm run dev
```

Vite is configured to bind to `0.0.0.0`, so you can also open the printed network URL on your iPhone (same Wi-Fi) and test it as a real PWA. To install to home screen on iOS, you need HTTPS — easiest options below.

## Test on your iPhone (free, no Apple Developer account)

**Option 1 — LAN with HTTPS via Caddy or a tunnel:**
- Run `npm run dev`, then expose it with `npx localtunnel --port 5173` or `cloudflared tunnel --url http://localhost:5173`
- Open the HTTPS URL on your iPhone in Safari
- Tap Share → Add to Home Screen

**Option 2 — Deploy to a free host (recommended for real use):**

```bash
npm run build
```

The `dist/` folder is a static site. Drop it on any of:

| Host | Cost | Notes |
|------|------|-------|
| **Cloudflare Pages** | Free | Fastest CDN, custom domain free, your usual stack |
| **Vercel** | Free | Easiest deploy, automatic HTTPS, `vercel` CLI |
| **GitHub Pages** | Free | Fine for static; needs base path tweak if not deploying to root |
| **Netlify** | Free | Drag-and-drop `dist/` into the dashboard |

Once deployed:
1. Open the site in **Safari** on your iPhone (must be Safari, not Chrome, for proper install)
2. Tap the Share button
3. Tap **Add to Home Screen**
4. The icon appears like a native app, opens fullscreen, no browser chrome

## Architecture

```
index.html      → semantic shell, links manifest + SW
src/style.css   → EasyFrame dark theme (Inter, blue/cyan accents)
src/main.js     → all app logic (no framework)
public/
  manifest.webmanifest  → PWA install metadata
  sw.js                  → service worker (offline cache)
  icons/                 → app icons (auto-generated by scripts/make_icons.py)
```

**Photo pipeline:** HTMLImageElement → compute frame dimensions → fill canvas with border color → draw image at offset → `toBlob('image/png')` or `toBlob('image/jpeg')` based on quality.

**Video pipeline:** HTMLVideoElement → drawImage onto an offscreen canvas every frame → `canvas.captureStream()` + original audio track → `MediaRecorder` → Blob. MP4 (H.264) when the browser supports it, falls back to WebM.

**Why no framework?** The state surface is tiny and the work is all imperative canvas painting. React would add weight, not value.

## Browser support

- iOS Safari 16.4+ (Web Share API with files, MediaRecorder for video export)
- Chrome / Edge: full support including MP4 H.264 encoding
- Older iOS Safari: photos work; video export falls back to WebM and downloads instead of share
